247 // The following 3 macros ending with TOL should have a tolerance
248 // passed to them. This tolerance should be something like
249 // g_triangletolerancexy, or a tolerance that has been computed
250 // from the range of the numbers involved. The numbers are compared
251 // against the tolerance
252 // Equal-with-tolerance EQ_TOL is needed, otherwise double values would not be
253 // equal when, for all practical purposes, they are equal. Given that a number
254 // is equal-with-tolerance to another number, then it cannot also be either
255 // less-than or greater-than that other number. Therefore, LT_TOL and GT_TOL
256 // give non-overlapping results rather than the result given by < and >.
